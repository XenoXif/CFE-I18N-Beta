Gendered Dialogue

Changes for use with GNT mod (i've added all the steps so that you can decide what you're able to do, i will handle what you can't):
 
 available config from GNT:
 "Singular": false,
  "SubjectivePronoun": "They",
  "ObjectivePronoun": "Them",
  "PosessivePronoun": "Their",
  "Title": "Mx.",
  "Adjective": "Gorgeous",
  "SpouseNoun": "Partner",
  "GenericNoun": "Person",
  "AdultAntiquatedNoun": "Folk",
  "AdultFamilialNoun": "Kid",
  "AdultInformalNoun": "Person",
  "ElderAntiquatedNoun": "Kid",
  "ElderInformalNoun": "Kid",
  "ElderFamilialNoun": "Child",
  "ChildFamilialNoun": "Cousin",
  "ServiceFormalNoun": "Esteemed Patron",
  "TraderNoun": "Traveller",
  "ParentalNoun": "Lala"

  in default.json:
  erase the second line after the ^
  replace the gendered portion of dialogue with the relevant config option above, in {{double brackets}}.
  leave a //comment with what you want the male and female standard lines to be.

  example: "Are you for real? You're the kind of guy who just shows up with free cake?$h^Are you for real? You're the kind of girl who just shows up with free cake?$h#$b#*sniff* Mmmm, and the best kind of cake, at that!" -->
  :"Are you for real? You're the kind of {{GenericNoun}} who just shows up with free cake?$h" //guy^girl

  in npc.json:
  find the appropriate key, and add: |defaultstoken={{configvaluefromgnmt}}}}" to the end of the i18n token.

  example: "AcceptGift_(O)220": "{{i18n:CDE_Abigail_ChocolateCake |GenericNoun={{GenericNoun}}}}"

  in content.json: 
  make a dynamic token that adds the standard gendered dialogue for when people are not using GNTM.
  A different dynamic token entry will need to be made for each of the config options from GNTM that you use (ie subjective, objective, possessive, etc)
  the way it is currently set up, i can't easily allow for multiple generic nouns for the male/female players, so if you for example are coding for "lad^lass" instead of "guy^girl", it will be easier to choose a different config key such as AdultInformalNoun rather than trying to fit multiple options into the GenericNoun category. 
  example: 
  "DynamicTokens": [
  { "Name": "GenericNoun",
	"Value": "guy",
	"When": { "PlayerGender": "male"}
	},
 { "Name": "GenericNoun",
	"Value": "girl",
	"When": { "PlayerGender": "female"}
	},
 { "Name": "GenericNoun"
   "Value": "{{Lowercase: {{Random: {{Hana.GNMTokens/GenericNoun}}}}}}",
   "When": {"HasMod": "Hana.GNMTokens"}
    }
  ]

  in manifest.json:
  make gnmtokens a false dependency!
